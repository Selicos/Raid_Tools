# v0.8 Development Plan

**Branch:** v0.8  
**Start Date:** 2025-11-03  
**Status:** In Progress  

## Overview

Version 0.8 focuses on three major improvements to the Raid Tools project:

1. **Champion Summary Generator** - Convert JSON entries to readable markdown with AI-generated analysis
2. **Gear Inventory Index** - Build comprehensive gear database with optimization tools
3. **Complete Champion Dictionary** - Finish remaining owned champion JSON entries

## Goals

### Goal 1: Champion Summary Generator with AI Analysis
**Purpose:** Create human-readable champion summaries from JSON dictionary entries, enhanced with Copilot-generated strategic analysis.

**Deliverables:**
- Champion summary markdown template (`input/Templates/Champion_Summary_Template.md`)
- Python script to convert JSON → Markdown (`Tools/champion_summary_generator.py`)
- AI analysis integration for mechanics, use cases, gear, and team recommendations
- Output directory: `Output/Champion_Summaries/`

**Why:** Current JSON entries are comprehensive but not optimized for quick reference. Markdown summaries with AI analysis will provide actionable insights for team building and gear optimization.

### Goal 2: Gear Inventory Index & Optimization
**Purpose:** Track all owned gear with stats, substats, awakening bonuses, and location (equipped/vault). Enable gear-based champion optimization.

**Deliverables:**
- Gear inventory JSON schema (`input/Templates/Gear_Inventory_Schema.json`)
- Gear indexing tool (`Tools/gear_indexer.py`)
- Gear recommendation engine (`Tools/gear_optimizer.py`)
- Centralized gear database (`input/Gear_Inventory/gear_index.json`)

**Why:** Currently, gear optimization requires manual comparison. A searchable gear database will enable automated recommendations for champion builds based on available inventory.

### Goal 3: Complete Remaining Champion Dictionary
**Purpose:** Finish all incomplete champion JSON entries, prioritizing owned champions.

**Deliverables:**
- List of remaining incomplete champions
- Completed JSON entries for all owned champions (Owned > 0 in Champion_stats.md)
- All entries moved to `input/Champion_Dictionary/Complete/`

**Why:** Complete champion dictionary enables accurate team building, gear optimization, and strategic planning across all content.

- **Status:** Not Started
- **Description:** Create JSON schema for gear indexing at `input/Templates/Gear_Inventory_Schema.json`
- **Fields:**
  - `gear_id` (string, unique identifier)

  - `awakening_bonuses` (array of objects: [{stat: string, value: number}])
  - `equipped_champion` (string or null)
  - `location` (enum: champion/vault)

#### Task 5: Build Gear Recommendation Engine
- **Status:** Not Started
- **Description:** Create tool to recommend gear upgrades for champions
- **Input:** Champion name + target content (CB/Dungeons/etc.)
- **Script Location:** `Tools/gear_optimizer.py`
- **Logic Flow:**
  1. Read champion stat priorities from JSON (stat_priority_recommendations section)
  2. Query `gear_index.json` for unequipped gear
  3. Calculate stat improvements for each potential swap
  4. Rank recommendations by impact (weighted by priority stats)
  5. Consider set bonuses and synergies
- **Output:** Markdown report at `Output/Gear_Recommendations/{Champion_Name}_{Content}_Recommendations.md`
- **Features:**
  - Top 5 gear swap recommendations
  - Expected stat changes
  - Set bonus analysis
  - EHP calculations
- **Dependencies:** Task 4 (gear indexer must exist), populated gear_index.json

## Notes & Decisions

### Design Decisions
- **Champion Summaries:** Markdown format chosen for readability and git-friendly versioning
- **Gear Index:** Centralized JSON chosen over individual files for easier querying
- **AI Analysis:** Placeholder approach allows manual editing before full automation

### Open Questions
- **Gear OCR:** Should we invest in screenshot parsing for v0.8 or defer to v0.9?
  - **Decision:** Defer to v0.9, use manual entry for v0.8
- **Summary Automation:** Should summaries auto-regenerate on JSON updates?
  - **Decision:** Manual regeneration for v0.8, automation in v0.9

### Risks & Mitigation
- **Risk:** Gear indexing is manual and time-consuming
  - **Mitigation:** Start with 20-piece sample, expand incrementally
- **Risk:** AI analysis quality varies by champion complexity
  - **Mitigation:** Create detailed prompt templates, validate outputs
- **Risk:** Champion completion may take longer than estimated
  - **Mitigation:** Use batch scraper, focus on owned champions only
